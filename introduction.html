artemis-odb / ECS

  • a World instance runs the simulation
    • manages lifecycle of entities + components
    • runs game loop (World::process) via Systems
  • provides serialization: full scene, entity subset, and Prefab
  • cuts down on boilerplate via dependency-injection, implicit and explicit

artemis-odb / Entity

  • container of related Component:s
  • represented as integer id
    • id:s starts at 0
    • id:s of deleted entities are recycled
  • World::create() for new entities
  • also: World::delete(entityId)

artemis-odb / Component

  • pure data classes: 
public class Position extends Component {
    public final Vector2 xy = new Vector2();
}
  • eg, Position, Size, FontReference
    • create/get/set via ComponentMapper<T>
    • Entity can only have one of each component type

artemis-odb / System

  • game logic building blocks
  • typically acts on a family of components, via Aspect:s
    • eg Aspect.all(Position.class, Size.class).exclude(Culled.class)

artemis-odb / Aspect

  • matches Entities based on a Component pattern
  • Often used in conjunction with AspectSubscriptionManager:
retrieve entites having a specific set of components
IntBag entityIds = world.getAspectSubscriptionManager()
    .get(all(Position.class, Size.class))
    .getEntities();
  • tip: import static com.artemis.Aspect.*

artemis-odb / Aspect

listen in on new or removed entities
world.getAspectSubscriptionManager()
    .get(aspect)
    .addSubscriptionListener(
        new EntitySubscription.SubscriptionListener() {
            @Override public void inserted(IntBag entities) {}
            @Override public void removed(IntBag entities) {}
})

artemis-odb / World

Orion / operations

  • composable actions for entities
  • respects entity lifecycle
  • (for the most part) serializable, incl resuming from serialized running state. doesn’t play nice with run and similar tricky operations.
  • 3 main base types: raw Operation, SinglueUseOperation and TemporalOperation
  • Example operations: moveTo, moveBy, directSreen, sizeTo, loadAsset

Orion / operations

operation exhibit
sequence(                                           <1>
    delay(seconds(1 / 6f)),                         <2>
    parallel(                                       <3>
        alpha(1f, seconds(2.5f), sineOut),          <4>
        moveBy(xy(10, 0), seconds(1.65f), swingOut)
    ),
    log("last operation in sequence()")
).register(e);                                      <5>
<1> `sequence` and `parallel`; primary means of composing operations
<2> `import static UrizenOperations.\*` and `import static OperationFactory.*`
<3> `parallel` runs until all child operations have completed
<4> `TemporalOperation` types typically support interpolation (`com.badlogic.gdx.math.Interpolation`)
<5> Runs operation on entity `e`. Operations are single use / consumed when registering.

Common operations

fork

Runs Operation on a different entity.

sequence(
    fork(id1, removeComponent(Culled.class)),
    fork(id2, removeComponent(Culled.class)),
    fork(playerTag,
        sequence(
            delay(seconds(3f)),
            removeComponent(Culled.class),
            sizeTo(xy(2f, -12.4f))))
).register(world);

Common operations

unique (per entity)
  • only a single non-completed unique per tag
  • adding another unique with the same tag completes the previous one
  • primarily used for cancelling conflicting or invalid operations 
sequence(
    unique("lock", scoreBlock()), // blocks until full score
    updateThemeArrow()
).register(nextTheme);

Director/DirectedScreen

  • Director runs a DirectedScreen
    • DirectedScreen::render in turn calls its World::process
  • provides DirectedScreen::onEnter and DirectedScreen::onExit
  • "loading" screen is special
    • only shown when assets require loading
    • configure with loading.enter.fadeIn, loading.exit.fadeOut, loading.black
    • unloading is explicit / by the user
fade out and into "map" screen
factory.overlayFadeOut(camera,
    directScreen("map", new ScreenParameters()
        .set("theme.id", app().theme().themeId)
        .set("loading.enter.fadeIn", true)
        .set("loading.exit.fadeOut", true)));

<1> see directScreen("map", screenParams)
<2> required parameters, unless @Parameter(required = false)
<3> relays via "loading" screen, if any dependencies are not loaded
<4> parameter substitution for paths
<5> Asset::type defaults to AssetBundle.class

Director/DirectedScreen

Screens must be declared prior to initializing the world.

director = new Director(
    LoadingScreen.class,
    BootLoadingScreen.class,
    TitleScreen.class,
    MapScreen.class,
    CrozzleGameScreen.class);

MapApplication application = new MapApplication(-1, director, new DesktopAppRate());
new LwjglApplication(application, config);
